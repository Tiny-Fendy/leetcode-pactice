# 力扣算法题练习

> 本仓库用于 LeetCode 算法题目的练习，采用 TypeScript + Vitest 测试框架。

## 📋 Agent 工作指南

### 创建新题目

当用户要求创建新题目时，按以下步骤操作：

1. **创建题目目录**
   ```bash
   mkdir -p "src/[题号].[题目名称]"
   ```
   - 题目名称中的空格需要保留（如 `76. 最小覆盖子串`）
   - 使用中文题目名称

2. **创建解答文件 `[题号].ts`**
   ```typescript
   /**
    * [题号]. [题目名称]
    * https://leetcode.cn/problems/[英文slug]/
    * 
    * [题目描述]
    * 
    * 示例:
    * [示例输入输出]
    */

   function [函数名](参数): 返回类型 {
       // TODO: 实现你的算法
       
       return 默认值;
   }

   export default [函数名];
   ```
   - **只创建题目模板，不要直接写答案**
   - 包含完整的题目描述、链接和示例
   - 添加 TODO 注释提示待实现
   - 使用 `export default` 导出函数

3. **创建测试文件 `[题号].test.ts`**
   ```typescript
   import { describe, it, expect } from 'vitest'
   import [函数名] from './[题号]'

   describe('[题号]. [题目名称]', () => {
     it('示例 1: [描述]', () => {
       expect([函数名]([参数])).toBe(期望值)
     })

     // 补充更多测试用例...
   })
   ```
   - **必须补充丰富的测试用例**（至少 8-12 个）
   - 覆盖：题目示例、边界情况、特殊情况
   - 从 leetcode 官方测试用例中获取更多案例

4. **验证测试**
   ```bash
   npm test -- src/[题号].* --run
   ```

### 更新已有题目

- **补充测试用例**：直接在 `.test.ts` 文件中添加新的 `it()` 块
- **添加题后总结**：创建 `README.md`，记录知识点、思路、关键点、易错点
- **优化算法**：更新 `.ts` 文件，同时更新 `README.md` 记录优化对比

### 提交代码

```bash
git add "src/[题号].*/"
git commit -m "[简洁的提交信息]"
git push
```

## 📁 项目结构

```
src/
├── [题号].[题目名称]/
│   ├── [题号].ts        # 题目解答（留空实现，供用户练习）
│   ├── [题号].test.ts   # 测试用例（需补充完整）
│   └── README.md        # 题后总结（可选，记录解题思路）
```

### 目录特点

1. **每题独立目录**：每个题目有独立的文件夹，便于管理
2. **文件名简洁**：使用题号作为文件名，避免中文文件名的问题
3. **测试驱动**：先写测试用例，再实现算法
4. **题后总结**：可选的 README.md 记录解题思路和 optimization

## 🧪 运行测试

```bash
npm test              # 运行所有测试（监听模式）
npm test -- --run     # 运行一次不监听
npm test -- src/[题号]* --run  # 运行特定题目测试
npm run test:coverage # 生成覆盖率报告
npm run test:ui       # UI 界面运行测试
```

## 🛠️ 技术栈

- **TypeScript** - 类型安全的 JavaScript
- **Vitest** - 快速的 Vite 原生测试框架
- **Node.js** - 运行时环境

## 📝 开发规范

1. **解答留白**：创建题目时只提供模板，解答留给用户完成
2. **测试完整**：测试用例要覆盖各种场景，确保能验证正确性
3. **命名一致**：函数名与 LeetCode 官方保持一致
4. **导出方式**：统一使用 `export default`
