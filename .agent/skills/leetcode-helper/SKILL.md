# LeetCode Helper Skill

本 skill 用于管理 LeetCode 算法题练习仓库，确保按照统一规范创建和维护题目。

## 核心原则

1. **解答留白**：创建题目时只提供模板和题目描述，不直接写答案
2. **测试完整**：必须补充丰富的测试用例（至少 8-12 个）
3. **目录规范**：每题独立目录，文件名使用题号
4. **导出统一**：使用 `export default` 导出函数

## 工作流程

### 创建新题目

当用户要求创建新题目时，按以下步骤操作：

#### 1. 创建题目目录

```bash
mkdir -p "src/[题号].[题目名称]"
```

**注意**：
- 题目名称中的空格需要保留（如 `76. 最小覆盖子串`）
- 使用中文题目名称

#### 2. 创建解答文件 `[题号].ts`

```typescript
/**
 * [题号]. [题目名称]
 * https://leetcode.cn/problems/[英文slug]/
 * 
 * [题目描述]
 * 
 * 示例:
 * [示例输入输出]
 */

function [函数名](参数): 返回类型 {
    // TODO: 实现你的算法
    
    return 默认值;
}

export default [函数名];
```

**关键点**：
- 包含完整的题目描述、链接和示例
- 添加 TODO 注释提示待实现
- **不要直接写答案**，留给用户练习

#### 3. 创建测试文件 `[题号].test.ts`

```typescript
import { describe, it, expect } from 'vitest'
import [函数名] from './[题号]'

describe('[题号]. [题目名称]', () => {
  it('示例 1: [描述]', () => {
    expect([函数名]([参数])).toBe(期望值)
  })

  it('示例 2: [描述]', () => {
    expect([函数名]([参数])).toBe(期望值)
  })

  // 补充更多测试用例...
})
```

**测试用例要求**（至少 8-12 个）：
- ✅ 题目官方示例（2-3 个）
- ✅ 边界情况（空输入、单个元素等）
- ✅ 特殊情况（全部相同、全部不同等）
- ✅ 复杂情况（多个可能答案等）

#### 4. 验证测试

```bash
npm test -- src/[题号]* --run
```

确保测试文件语法正确，预期值合理。

### 更新已有题目

#### 补充测试用例
直接在 `.test.ts` 文件中添加新的 `it()` 块，确保覆盖更多场景。

#### 添加题后总结
创建 `README.md`，记录：
- 知识点
- 解题思路
- 关键实现要点
- 易错点
- 复杂度分析
- 优化方向（如有）

#### 优化算法
更新 `.ts` 文件后，同步更新 `README.md` 记录优化对比。

### 提交代码

```bash
git add "src/[题号].*/"
git commit -m "[简洁的提交信息]"
git push
```

## 项目结构

```
src/
├── [题号].[题目名称]/
│   ├── [题号].ts        # 题目解答（留空实现）
│   ├── [题号].test.ts   # 测试用例（需完整）
│   └── README.md        # 题后总结（可选）
```

## 常用命令

```bash
npm test                    # 运行所有测试（监听模式）
npm test -- --run           # 运行一次不监听
npm test -- src/[题号]* --run  # 运行特定题目
npm run test:coverage       # 生成覆盖率报告
```

## 注意事项

1. **不要覆盖用户的解答**：如果用户已经实现了算法，只更新测试或 README
2. **测试用例要准确**：从 LeetCode 官方获取正确的预期值
3. **保持命名一致**：函数名与 LeetCode 官方保持一致
4. **目录名保留空格**：如 `76. 最小覆盖子串` 而不是 `76.最小覆盖子串`
