# 题后总结

## 知识点
排序 + 贪心算法

## 思路
1. **排序**：按区间的起始位置升序排序，确保处理时区间是有序的
2. **遍历合并**：逐个处理区间，与结果数组的最后一个区间比较
3. **判断重叠**：
   - 如果不重叠（`last[1] < cur[0]`）：直接加入结果数组
   - 如果重叠：更新最后一个区间的终点为两者终点的最大值
4. **返回结果**：遍历完成后，结果数组即为合并后的区间

## 关键点
1. **原地修改**：直接修改 `result` 中最后一个区间的终点值，而不是创建新区间
2. **边界判断**：`last[1] < cur[0]` 表示不重叠（注意是严格小于，等于也算重叠）
3. **取最大值**：合并时使用 `Math.max(last[1], cur[1])`，防止小区间被大区间包含的情况
4. **空结果处理**：第一个区间直接 push，无需比较

## 核心逻辑
```typescript
if (last[1] < cur[0]) {
    // 不重叠，直接添加
    result.push(cur);
} else {
    // 重叠，更新终点
    last[1] = Math.max(last[1], cur[1]);
}
```

## 易错点
1. **排序的重要性**：必须先按起点排序，否则无法保证正确性
2. **重叠判断条件**：`last[1] < cur[0]` 是不重叠，`>=` 才是重叠（包括端点相等的情况）
3. **更新终点而非起点**：合并时只更新终点，起点保持不变（因为已排序，当前区间的起点一定 >= 上一个区间的起点）
4. **原地修改 vs 新建**：使用 `last[1] = Math.max(...)` 原地修改，而不是 `result.push([last[0], Math.max(...)])`

## 复杂度
- 时间复杂度：O(n log n)，排序占主导，遍历为 O(n)
- 空间复杂度：O(1)（不考虑输出数组），排序可能占用 O(log n) 栈空间

## 示例演示
以 `intervals = [[1,3],[2,6],[8,10],[15,18]]` 为例：

**排序后**：`[[1,3],[2,6],[8,10],[15,18]]`（已有序）

| i | cur | last | 操作 | result |
|---|-----|------|------|--------|
| 0 | [1,3] | undefined | 直接 push | [[1,3]] |
| 1 | [2,6] | [1,3] | 3 >= 2，重叠，更新终点 max(3,6)=6 | [[1,6]] |
| 2 | [8,10] | [1,6] | 6 < 8，不重叠，push | [[1,6],[8,10]] |
| 3 | [15,18] | [8,10] | 10 < 15，不重叠，push | [[1,6],[8,10],[15,18]] |

最终结果：`[[1,6],[8,10],[15,18]]`
