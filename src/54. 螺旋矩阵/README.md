# 题后总结

## 知识点
模拟遍历 + 方向数组 + 步数递减

## 思路（优化解法）
1. **核心思想**：用方向数组 `DES` 控制移动，每走完一个方向后减少对应维度的步数
2. **方向数组**：`[[1,0], [0,1], [-1,0], [0,-1]]` 分别代表右、下、左、上
3. **步数规律**：
   - 横向移动（右/左）：步数为 `colLen`，走完后 `colLen--`
   - 纵向移动（下/上）：步数为 `rowLen`，走完后 `rowLen--`
4. **循环终止**：当结果数组长度等于矩阵总元素数时结束

## 核心逻辑
```typescript
const DES = [[1, 0], [0, 1], [-1, 0], [0, -1]]; // 右、下、左、上

let colLen = matrix[0].length;  // 横向长度
let rowLen = matrix.length - 1; // 竖向长度（注意减1）
let SIZE = colLen * (rowLen + 1);
let loc = [-1, 0]; // 起始位置（-1, 0），第一次移动后变为 (0, 0)

const result: number[] = [];

for (let di = 0; result.length < SIZE; di = (di + 1) % 4) {
    const [dc, dr] = DES[di];
    const step = dc === 0 ? rowLen : colLen; // 根据方向决定步数

    for (let i = 0; i < step; i++) {
        loc[0] += dc;
        loc[1] += dr;
        result.push(matrix[loc[1]][loc[0]]);
    }

    dc === 0 ? rowLen-- : colLen--; // 收缩对应维度
}
```

## 关键点
1. **方向数组**：用二维数组表示四个方向的增量，通过 `di % 4` 循环切换
2. **起始位置**：`loc = [-1, 0]`，这样第一次向右移动时从 `(0, 0)` 开始
3. **步数计算**：
   - `dc === 0` 表示纵向移动（下/上），步数为 `rowLen`
   - `dc !== 0` 表示横向移动（右/左），步数为 `colLen`
4. **步数递减**：每走完一个方向，对应维度的长度减 1

## 易错点
1. **rowLen 初始值**：必须是 `matrix.length - 1`，因为第一行已经算在 colLen 里了
2. **起始位置**：`[-1, 0]` 而不是 `[0, 0]`，确保第一次移动从左上角开始
3. **步数判断**：`dc === 0` 判断纵向，不是 `dr === 0`
4. **循环条件**：用 `result.length < SIZE` 而非方向计数，更直观

## 复杂度
- 时间复杂度：O(m × n)，每个元素访问一次
- 空间复杂度：O(1)（不计输出数组）

## 示例演示
以 `matrix = [[1,2,3],[4,5,6],[7,8,9]]` 为例：

初始：`colLen=3, rowLen=2, SIZE=9, loc=[-1,0]`

| 方向 di | [dc,dr] | step | 移动过程 | loc 变化 | result | 更新后 |
|---------|---------|------|---------|----------|--------|--------|
| 0 (右) | [1,0] | 3 | (0,0)→(0,1)→(0,2) | [-1,0]→[2,0] | [1,2,3] | colLen=2 |
| 1 (下) | [0,1] | 2 | (1,2)→(2,2) | [2,0]→[2,2] | [1,2,3,6,9] | rowLen=1 |
| 2 (左) | [-1,0] | 2 | (1,2)→(0,2) | [2,2]→[0,2] | [1,2,3,6,9,8,7] | colLen=1 |
| 3 (上) | [0,-1] | 1 | (0,1) | [0,2]→[0,1] | [1,2,3,6,9,8,7,4] | rowLen=0 |
| 0 (右) | [1,0] | 1 | (1,1) | [0,1]→[1,1] | [1,2,3,6,9,8,7,4,5] | colLen=0 |

最终结果：`[1,2,3,6,9,8,7,4,5]` ✓
