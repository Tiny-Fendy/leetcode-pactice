# 题后总结

## 知识点
原地哈希（In-place Hashing）+ 数组置换

## 思路
1. **核心思想**：将每个正整数 `x` 放到索引 `x-1` 的位置上，即 `nums[x-1] = x`
2. **第一遍遍历**：对每个位置执行原地置换，直到所有能归位的正整数都在正确位置
3. **第二遍遍历**：找到第一个 `nums[i] !== i + 1` 的位置，返回 `i + 1`
4. **如果都正确**：说明 `[1, n]` 都在数组中，返回 `n + 1`

## 核心逻辑
```typescript
// 原地置换
for (let i = 0; i < nums.length; i++) {
    while (
        nums[i] > 0 &&                    // 是正整数
        nums[i] !== i + 1 &&              // 不在正确位置
        nums[nums[i] - 1] < nums.length && // 目标位置有效
        nums[nums[i] - 1] !== nums[i]     // 避免重复数字死循环
    ) {
        [nums[nums[i] - 1], nums[i]] = [nums[i], nums[nums[i] - 1]];
    }
}

// 查找缺失的正数
for (let i = 0; i < nums.length; i++) {
    if (nums[i] !== i + 1) {
        return i + 1;
    }
}
return nums.length + 1;
```

## 关键点
1. **while 循环条件**（四个条件缺一不可）：
   - `nums[i] > 0`：只处理正整数
   - `nums[i] !== i + 1`：当前数字不在正确位置
   - `nums[nums[i] - 1] < nums.length`：目标位置的值在有效范围内（防止越界和无限循环）
   - `nums[nums[i] - 1] !== nums[i]`：避免重复数字导致死循环

2. **交换顺序**：解构赋值 `[nums[nums[i] - 1], nums[i]] = [nums[i], nums[nums[i] - 1]]` 同时完成两个位置的交换

3. **时间复杂度保证**：虽然嵌套了 while 循环，但每个元素最多被交换到正确位置一次，总体仍是 O(n)

## 易错点
1. **重复数字处理**：如果没有 `nums[nums[i] - 1] !== nums[i]` 条件，遇到重复数字会无限循环
2. **边界条件**：`nums[nums[i] - 1] < nums.length` 确保目标值不会导致后续访问越界
3. **交换顺序**：解构赋值时，左边先计算 `nums[nums[i] - 1]`，此时 `nums[i]` 还是旧值
4. **返回值**：如果所有位置都正确，返回 `nums.length + 1`

## 复杂度
- 时间复杂度：O(n)，每个元素最多被交换一次
- 空间复杂度：O(1)，原地操作，只使用常数额外空间

## 示例演示
以 `nums = [3, 4, -1, 1]` 为例：

**第一遍遍历（原地置换）**：

| i | nums[i] | 操作 | nums |
|---|---------|------|------|
| 0 | 3 | 3 应该在索引 2，交换 nums[0] 和 nums[2] | [-1, 4, 3, 1] |
| 0 | -1 | 负数，跳过 | [-1, 4, 3, 1] |
| 1 | 4 | 4 应该在索引 3，交换 nums[1] 和 nums[3] | [-1, 1, 3, 4] |
| 1 | 1 | 1 应该在索引 0，交换 nums[1] 和 nums[0] | [1, -1, 3, 4] |
| 1 | -1 | 负数，跳过 | [1, -1, 3, 4] |
| 2 | 3 | 3 在正确位置（索引 2），跳过 | [1, -1, 3, 4] |
| 3 | 4 | 4 在正确位置（索引 3），跳过 | [1, -1, 3, 4] |

**第二遍遍历（查找缺失）**：

| i | nums[i] | 期望值 i+1 | 是否匹配 |
|---|---------|-----------|---------|
| 0 | 1 | 1 | ✓ |
| 1 | -1 | 2 | ✗ → 返回 2 |

最终结果：`2` ✓

## 算法本质

这道题的本质是**用数组本身作为哈希表**：
- 传统哈希表：用额外空间记录哪些数字出现过
- 原地哈希：利用数组索引与值的对应关系，`nums[i] = i + 1` 表示数字 `i + 1` 出现过

## 扩展思考
- 这是"原地哈希"的经典题目，类似思路可用于其他需要 O(1) 空间的数组问题
- 相关题目：**448. 找到所有数组中消失的数字**、**442. 数组中重复的数据**
