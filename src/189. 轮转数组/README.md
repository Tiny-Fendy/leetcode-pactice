# 题后总结

## 知识点
数组操作 + 原地修改

## 思路
1. **边界处理**：k 等于数组长度时直接返回；k 大于数组长度时取模 `k = k % nums.length`
2. **保存尾部元素**：使用 `slice(-k)` 截取最后 k 个元素，这些元素会移动到数组开头
3. **从后往前移动**：避免覆盖未处理的元素
   - 索引 `i > k-1` 的元素：向后移动 k 位，`nums[i] = nums[i - k]`
   - 索引 `i <= k-1` 的元素：用保存的尾部元素填充，`nums[i] = tem[i]`
4. **原地修改**：直接修改原数组，不返回新数组

## 关键点
1. **从后往前遍历**：防止前面的元素被覆盖后丢失
2. **分界点 `k-1`**：
   - `i > k-1`：原始数组中的元素向后移动
   - `i <= k-1`：用截取的尾部元素填充
3. **空间换时间**：使用 `tem` 数组保存尾部 k 个元素，空间复杂度 O(k)

## 核心逻辑
```typescript
const tem = nums.slice(-k);  // 保存尾部 k 个元素

for (let i = nums.length - 1; i >= 0; i--) {
    if (i > k - 1) {
        nums[i] = nums[i - k];  // 后半部分向后移
    } else {
        nums[i] = tem[i];       // 前半部分用尾部元素填充
    }
}
```

## 易错点
1. **k 取模**：k 可能大于数组长度，必须先 `k = k % nums.length`
2. **遍历方向**：必须从后往前，否则前面的元素会被覆盖
3. **分界条件**：`i > k - 1` 而不是 `i >= k`，确保正确划分前后两部分
4. **原地修改**：不能创建新数组返回，必须修改原数组

## 复杂度
- 时间复杂度：O(n)，遍历一次数组
- 空间复杂度：O(k)，存储尾部 k 个元素

## 示例演示
以 `nums = [1,2,3,4,5,6,7], k = 3` 为例：

**步骤 1**：`tem = nums.slice(-3) = [5, 6, 7]`

**步骤 2**：从后往前遍历

| i | 条件 | 操作 | nums |
|---|------|------|------|
| 6 | i > 2 | nums[6] = nums[3] = 4 | [1,2,3,4,5,6,4] |
| 5 | i > 2 | nums[5] = nums[2] = 3 | [1,2,3,4,5,3,4] |
| 4 | i > 2 | nums[4] = nums[1] = 2 | [1,2,3,4,2,3,4] |
| 3 | i > 2 | nums[3] = nums[0] = 1 | [1,2,3,1,2,3,4] |
| 2 | i <= 2 | nums[2] = tem[2] = 7 | [1,2,7,1,2,3,4] |
| 1 | i <= 2 | nums[1] = tem[1] = 6 | [1,6,7,1,2,3,4] |
| 0 | i <= 2 | nums[0] = tem[0] = 5 | [5,6,7,1,2,3,4] |

最终结果：`[5,6,7,1,2,3,4]` ✓

## 其他解法对比

| 方法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 当前解法（切片+移位） | O(n) | O(k) | 直观易懂 |
| 三次反转 | O(n) | O(1) | 最优空间复杂度 |
| 环形替换 | O(n) | O(1) | 较复杂但高效 |
| 额外数组 | O(n) | O(n) | 最简单但空间开销大 |
