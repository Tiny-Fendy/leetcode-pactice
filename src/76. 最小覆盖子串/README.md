# 题后总结

## 知识点
滑动窗口 + 双数组频次统计 + 计数器优化

## 原始思路（双数组 + isCover 遍历）
1. **双数组统计**：用两个长度为 128 的数组分别记录 t 和当前窗口中各字符的频次
2. **右边界扩展**：遍历 s，将右边界字符加入窗口的频次统计
3. **isCover 判断**：每次遍历整个数组比较，判断窗口是否覆盖 t
4. **左边界收缩**：当窗口包含 t 的所有字符时，尝试收缩左边界寻找更小的覆盖子串

## 优化思路（计数器替代 isCover 遍历）
1. **requiredCount**：记录 t 中有多少种不同的字符需要匹配
2. **winCount**：记录窗口内已经满足数量要求的字符种类数
3. **智能计数**：只有当 `cntS[code] === cntT[code]` 时才增加 winCount，避免重复计数
4. **精准判断**：用 `winCount === requiredCount` 替代遍历整个数组的 `isCover` 函数

## 优化对比
| 方面 | 原始方法 | 优化方法 |
|------|---------|---------|
| 判断方式 | 遍历 128 个元素比较 | 直接比较计数器 |
| 时间复杂度 | O(n * C) | O(n) |
| 代码复杂度 | 需要额外 isCover 函数 | 逻辑内联，更清晰 |
| 性能 | 每次 while 循环都要遍历 | O(1) 判断 |

## 关键点
1. **requiredCount 计算**：遍历 t 时，只有当 `cntT[code] === 0` 时才增加，确保统计的是字符种类数
2. **winCount 更新时机**：
   - 右边界加入：只有当 `cntS[code] === cntT[code]` 时 winCount++（刚好满足需求）
   - 左边界移除：只有当 `cntS[lCode] === cntT[lCode]` 时 winCount--（即将不满足需求）
3. **只统计目标字符**：只有 `cntT[code] > 0` 的字符才参与计数，忽略无关字符
4. **BASE_CODE 常量**：提取 `'A'.charCodeAt(0)` 为常量，提高代码可读性

## 易错点
1. winCount 的增减条件容易搞反：必须是"刚好满足"或"即将不满足"时才更新
2. 左边界收缩时，先判断再减少计数，顺序不能颠倒
3. 只有目标字符（`cntT[code] > 0`）才参与窗口计数
4. 截取子串时使用 `s.substring(l, r + 1)`

## 复杂度
- 时间复杂度：O(n)，n 为 s 的长度，消除了字符集常数因子
- 空间复杂度：O(C)，两个长度为 128 的数组

## 核心思想
用**计数器**替代**全量比较**，这是滑动窗口问题的经典优化技巧。类似思路也应用于 438 题（找到字符串中所有字母异位词）。
