# 题后总结

## 知识点
滑动窗口 + 双数组频次统计

## 思路
1. **双数组统计**：用两个长度为 128 的数组分别记录 t 和当前窗口中各字符的频次
2. **右边界扩展**：遍历 s，将右边界字符加入窗口的频次统计
3. **左边界收缩**：当窗口包含 t 的所有字符时，尝试收缩左边界寻找更小的覆盖子串
4. **更新结果**：在收缩过程中，持续更新最小覆盖子串

## 关键点
1. `isCover` 函数判断窗口是否覆盖 t：遍历比较两个数组，窗口中每个字符的数量都要 >= t 中的数量
2. 使用 `while` 循环收缩左边界，直到窗口不再覆盖 t
3. 更新最小字符串的条件：`r - l + 1 <= minStr.length || !minStr.length`
4. ASCII 码映射：使用 `charCodeAt(0) - 'A'.charCodeAt(0)` 将字符映射到数组索引（覆盖 A-Z 和 a-z）

## 易错点
1. 边界条件：s 长度小于 t 或 t 为空时直接返回空字符串
2. `isCover` 是遍历整个数组比较，时间复杂度较高（但字符集固定为 128，可视为 O(1)）
3. 收缩左边界时要正确减少对应字符的计数
4. 截取子串时使用 `s.substring(l, r + 1)`，注意右边界是开区间

## 复杂度
- 时间复杂度：O(n * C)，n 为 s 的长度，C 为字符集大小（128）
- 空间复杂度：O(C)，两个长度为 128 的数组

## 优化方向
可以参考 438 题的不定长滑动窗口思路，用单个数组 + 计数器来避免每次调用 `isCover` 遍历整个数组，将时间复杂度优化到 O(n)。
